# 工作流与招聘复盘看板可靠性设计

## 目标

岗位流程在工作流中确认后，候选人台账与招聘复盘看板使用同一份阶段配置。同步命令生成完整的 `招聘数据复盘.html`，打开后直接展示岗位专属复盘，不出现流程配置弹窗，也不提供会混淆数据来源的手动上传入口。

## 数据与文件边界

- `PIPELINE.json` 是岗位阶段名称与顺序的唯一配置来源。阶段 ID 必须连续；台账与看板中的规范值为 `编号-名称`，例如 `0-简历初筛`。
- `candidate-ledger.xlsx` 是候选人流程事实源。更新时允许输入裸阶段名或规范阶段名，脚本依据 `PIPELINE.json` 映射并只写入规范值；不存在或重名的阶段拒绝写入。
- 岗位目录生成 `招聘数据复盘.html`。`index.html` 不再作为工作流生成物或文档命令中的目标名称。
- 同步前保留目标 HTML 的 `.bak` 备份；`--force` 重建时同样先备份，防止丢失已同步页面。
- `CONTEXT.md` 记录最近同步时间、最近同步记录数和看板文件名，供下一轮工作读取。

## 看板模式

### 工作流同步模式

`sync-dashboard-data.mjs` 接收台账、岗位名和可选上下文文件：

1. 读取台账的阶段选项与候选人记录。
2. 将数据和阶段顺序写入 `招聘数据复盘.html`。
3. 在注入代码中将 `STAGES_CONFIG` 更新为岗位阶段，并关闭 `stageConfigModal`。
4. 隐藏上传相关按钮、文件输入和初始空态入口；页面标题更新为“岗位名｜招聘数据复盘”。
5. 显示“数据来自候选人台账；请通过工作流更新后重新同步”的只读提示。
6. 根据存在面试日期但没有面试反馈摘要的记录，显示“面试反馈待补”数量与候选人名单。
7. 将同步时间与记录数写回 `CONTEXT.md`（仅在显式提供或同目录可定位时）。

同步数据为空时不进入只读模式，也不关闭初始配置弹窗，避免把空看板误显示为已同步。

### 手动模板模式

模板文件仍保留完整上传与流程配置能力，便于单独使用。只有被工作流同步过的岗位看板才切换成只读复盘模式。

## 候选人更新接口

新增首选写法：候选人更新输入 `stage`、`status` 与 `fields`，其中 `status` 只能为“进行中、通过、终止”。

- `stage` 由流程配置规范化为 `编号-名称`。
- `status=终止` 时必须有终止原因；其他状态会清空终止原因。
- `fields` 只更新明确传入的台账列。
- 旧的 `decision`（`terminate`、`defer`、`ready_for_interview`）暂时保留，以避免已有自动化失效，但文档和帮助均推荐直接写入方式。

## 漏斗口径

保留当前累计漏斗算法：某阶段数表示当前主阶段位于该阶段或其后续阶段的候选人数量。看板将把这一口径放在漏斗标题附近，以清晰文字展示；不把“当前阶段人数”误称为漏斗转化人数。

## 命令行体验

四个脚本均支持 `--help` / `-h`，输出用途、必填参数、可选参数和示例：

- `create-role-ledger.mjs`
- `create-review-dashboard.mjs`
- `sync-dashboard-data.mjs`
- `update-candidate-ledger.mjs`

`create-review-dashboard.mjs --force` 允许重建既有看板并写出 `.bak`；未带 `--force` 时继续拒绝覆盖。

## 新模板接入

以用户提供的 `E:\codex\Recruitment data\index.html` 替换项目内 `workflow/templates/recruitment-review-dashboard.html`。接入后保留工作流模板所需的稳定 DOM ID：`stageConfigModal`、`fileInput`、主标题和上传入口；若新版模板命名不同，则同步脚本通过受控标记或兼容选择器处理，不修改候选人数据口径。

## 不在本轮实施

- 不新增“一键重建/清空岗位”脚本，避免误删真实候选人事实源。
- 不批量重命名既有 Markdown 文件，避免破坏 Skill、文档与路径引用。
- 不默认生成候选人 ID；候选人 ID 是否应与企业招聘系统一致，需要另行确认。
- 移动端适配仅在新模板缺失基础断点时补最小样式，不重做桌面复盘布局。

## 验收

- 同步后的 `招聘数据复盘.html` 不显示阶段配置弹窗和上传入口，标题包含岗位名。
- 任何同步数据均不能被默认阶段配置覆盖。
- 裸阶段名能映射为唯一的 `编号-名称`；未知或重名阶段报错。
- 直接“阶段 + 状态”写入与旧 decision 写入均受测试覆盖。
- `--help` 和 `--force` 行为有自动化测试。
- 有面试日期、无反馈摘要的候选人在同步后被标记为待补。
- 同步元数据写入 `CONTEXT.md`，完整测试集通过。
