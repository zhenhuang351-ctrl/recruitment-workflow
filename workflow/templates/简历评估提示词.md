# 简历评估提示词

开始前读取 `AGENTS.md`、当前岗位的 `ROLE_STANDARD.md`、`candidate-ledger.xlsx` 和同岗位已有候选人档案。仅处理已获得授权的简历文本或 PDF 提取文本。

## 工作步骤

1. 先检查岗位标准是否已确认；未确认时仅做结构化整理，不给匹配判断。
2. 先用姓名、当前公司、当前岗位和教育经历提示可能重复记录；不能确定时请招聘者确认，不要擅自合并。
3. 检查台账必填流程信息：**简历来源**和**简历收取时间**。材料或用户输入未提供时，先只问一个问题补齐缺失字段；来源可选内部人才库、员工推荐、招聘平台、猎头、官网投递、活动/社群或其他，收取时间使用 `YYYY-MM-DD`。未确认前不得编造，也不得写入台账。
4. 输出必须**结论先行**：第一行固定为 `姓名｜建议：优先沟通 / 需验证 / 暂缓关注｜能力证据得分：x/4｜证据覆盖率：x%`。建议、得分和覆盖率必须在任何详细经历前出现。
5. 紧接第一行写不超过三条的整体判断；所有尚缺或无法确认的信息必须以 `**待核实**：...` 的粗体格式呈现。
6. 再输出结构化简历：姓名、年龄、学历、总工作年限、对口岗位年限、对口项目年限、当前公司/岗位、Base、关键经历、项目、量化成果与信息缺口。没有写明的字段填“待验证”。
7. 按岗位能力维度生成证据卡。每一维必须包含：原文摘录、来源位置、判断说明、证据状态、验证问题。禁止把模型推断写成事实。
8. 单独核对年龄和年限资格规则：只按 `ROLE_STANDARD.md` 中已确认的范围与口径提示“匹配 / 待核实 / 待人工确认”；不自动终止流程。
9. 生成候选人电话沟通问题，优先问证据缺口、动机、Base、薪资、职级、Offer 与到岗时间。
10. 在招聘者确认后，创建/更新候选人 Markdown 档案及候选人台账的一行；记录简历收取时间、来源、阶段和备注。

## 固定输出结构

### 结论先行

```text
{{姓名}}｜建议：{{优先沟通/需验证/暂缓关注}}｜能力证据得分：{{x/4}}｜证据覆盖率：{{x%}}
整体判断：{{最多三条、只基于已出现的证据}}
**待核实**：{{缺口1；缺口2}}
```

### 结构化简历

使用字段表格呈现，并标记每个字段的来源或“待验证”。

### 候选人证据卡

对每个维度写：`优势匹配情况`、`原文摘录`、`来源位置`、`**待核实**`、`须注意的`、`建议电话沟通问题`。

### 台账拟写回内容

先展示待写入的候选人台账字段和候选人 ID；等待人工确认后再写回。台账不应保存简历原文，只保存结构化事实和简短证据摘要。
